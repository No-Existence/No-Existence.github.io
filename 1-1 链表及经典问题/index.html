LC141 环形链表

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false 。

  • 链表中节点的数目范围是 [0, 104]
  • -105 <= Node.val <= 105
  • pos 为 -1 或者链表中的一个 有效索引 。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/linked-list-cycle

思路一：哈希表

遍历链表所有结点，每个遍历到的结点都存放在HashSet中。如果有结点在HashSet中已存在，则表示是环形链表。

空间复杂度O(n)，时间复杂度O(n)

思路二：快慢指针

定义一个快指针fast和一个慢指针slow，都指向首元结点。每次fast前进两步，慢结点前进一步。如果快慢指针相遇，则表明是环形链表；如果快指针指空，则表明是线性链表。

空间复杂度O(1) ，时间复杂度O(n)

LC142 环形链表2

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。

说明：不允许修改给定的链表。

  • 链表中节点的数目范围在范围 [0, 104] 内
  • -105 <= Node.val <= 105
  • pos 的值为 -1 或者链表中的一个有效索引

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/linked-list-cycle-ii
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

思路一：哈希表

同LC141 略

思路二：快慢指针

在LC141的基础上，在快慢指针初次相遇的时候新开一个指针p，让p和slow都同时每次前进一步，相遇的位置即为环的起点。

空间复杂度O(1) ，时间复杂度O(n)

LC202 快乐数

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：

对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
如果 可以变为 1，那么这个数就是快乐数。
如果 n 是快乐数就返回 true ；不是，则返回 false 。

示例

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/happy-number

思路

把每次求平方和的结果看作是链表的一个结点，由此构成一个链表。

如果链表有环则表明不是快乐数。

LC61 旋转链表

给你一个链表的头节点 head ，旋转链表，将链表每个节点向右移动 k 个位置。

提示：

  • 链表中节点的数目在范围 [0, 500] 内
  • -100 <= Node.val <= 100
  • 0 <= k <= 2 * 109

思路

首尾连成环形链表，头指针前移k位，即后移n-k-1位。注意 k可能大于n。

LC19 删除链表的倒数第 N 个结点

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

进阶：你能尝试使用一趟扫描实现吗？

提示：

  • 链表中结点的数目为 sz
  • 1 <= sz <= 30
  • 0 <= Node.val <= 100
  • 1 <= n <= sz

思路一：存储结点

为了一次遍历，将结点存放在数组中

思路二：双指针

让p指向首元结点，q指向表头结点。当p向后走n步时，q开始以相同速率往后走。当p为空时，q所指的结点即为待删除结点的前一个结点

LC83 删除排序链表中的重复元素

存在一个按升序排列的链表，给你这个链表的头节点 head ，请你删除所有重复的元素，使每个元素 只出现一次 。

返回同样按升序排列的结果链表。

思路

设立两个指针p和q，q停留在每个数字首次出现的位置，p从q的下一个位置开始逐结点向前遍历。当p与q所指的值不同时，直接将q.next指向p，q后移至p。

LC82 删除排序链表中的重复元素 II

存在一个按升序排列的链表，给你这个链表的头节点 head ，请你删除链表中所有存在数字重复情况的节点，只保留原始链表中 没有重复出现 的数字。

返回同样按升序排列的结果链表。

提示：

  • 链表中节点数目在范围 [0, 300] 内
  • -100 <= Node.val <= 100
  • 题目数据保证链表已经按升序排列

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/remove-duplicates-from-sorted-list-ii

思路

设立两个指针p和q。对于每个数字，q指向其首次出现结点的前一个位置，p指向其首次出现的位置。分两种情况:

  1. 如果p结点和它下一个结点的值相同，则表明该数字对应的所有结点都需要删除，为此循环将p移到下一个数字首次出现的位置，q.next指向p。
  2. 否则该数字应保留，p和q都向后顺移一位

LC86 分割链表

给你一个链表的头节点 head 和一个特定值 x ，请你对链表进行分隔，使得所有 小于 x 的节点都出现在 大于或等于 x 的节点之前。

你应当 保留 两个分区中每个节点的初始相对位置。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/partition-list

思路

利用归并排序的思想，将小于x的结点放在链表small里，把大于等于x的结点放在链表large里，最后将这两个链表串起来。

* LC138 复制带随机指针的链表

给你一个长度为 n 的链表，每个节点包含一个额外增加的随机指针 random ，该指针可以指向链表中的任何节点或空节点。

构造这个链表的 深拷贝。 深拷贝应该正好由 n 个 全新 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 next 指针和 random 指针也都应指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。复制链表中的指针都不应指向原链表中的节点 。

random_index：随机指针指向的节点索引（范围从 0 到 n-1）；如果不指向任何节点，则为 null 。
你的代码 只 接受原链表的头节点 head 作为传入参数。

常见错误

链表结点计数错误

链表遍历结束条件错误

未考虑给定的链表结点个数不足或链表为空

多个指针遍历链表时忘记某个指针后移

反转部分链表后忘记反转段前后指针的修改

常见方法

  • 反转链表的常用模式代码

    ​ LC 206, LC92 ,LC 25 有共同的反转链表的逻辑

  • 增加表头结点减少特判

    ​ 如直接返回给定的head可能会需要很多特判，但增加表头结点newHead后只需调用newHead.next即可。例如 LC82 , LC92, LC25

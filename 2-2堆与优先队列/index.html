剑指 Offer 40 最小的k个数 [简单]

输入整数数组 arr ，找出其中最小的 k 个数。例如，输入4、5、1、6、2、7、3、8这8个数字，则最小的4个数字是1、2、3、4。

思路

维护一个容量为k的大顶堆，每次将数组元素放入堆中。如果堆已满，则判断堆顶元素是否比待放入元素大。若是，则取出堆顶元素并放入待放入元素。遍历完后，堆中的所有元素即为所求

LC1046 最后一块石头的重量 [简单]

有一堆石头，每块石头的重量都是正整数。

每一回合，从中选出两块 最重的 石头，然后将它们一起粉碎。假设石头的重量分别为 x 和 y，且 x <= y。那么粉碎的可能结果如下：

如果 x == y，那么两块石头都会被完全粉碎；

如果 x != y，那么重量为 x 的石头将会完全粉碎，而重量为 y 的石头新重量为 y-x。

最后，最多只会剩下一块石头。返回此石头的重量。如果没有石头剩下，就返回 0。

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/last-stone-weight

提示：

  • 1 <= stones.length <= 30
  • 1 <= stones[i] <= 1000

思路

优先队列。

* LC703 数据流中的第 K 大元素

设计一个找到数据流中第 k 大元素的类（class）。注意是排序后的第 k 大元素，不是第 k 个不同的元素。

请实现 KthLargest 类：

KthLargest(int k, int[] nums) 使用整数 k 和整数流 nums 初始化对象。

int add(int val) 将 val 插入数据流 nums 后，返回当前数据流中第 k 大的元素。

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/kth-largest-element-in-a-stream

思路

我们可以使用一个大小为 k 的优先队列来存储前 k 大的元素，其中优先队列的队头为队列中最小的元素，也就是第 k 大的元素。

在单次插入的操作中，我们首先将元素 val 加入到优先队列中。如果此时优先队列的大小大于 k，我们需要将优先队列的队头元素弹出，以保证优先队列的大小为 k。

LC373 查找和最小的K堆数字 [中等]

给定两个以升序排列的整数数组 nums1 和 nums2 , 以及一个整数 k 。

定义一对值 (u,v)，其中第一个元素来自 nums1，第二个元素来自 nums2 。

请找到和最小的 k 个数对 (u1,v1), (u2,v2) … (uk,vk) 。

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/find-k-pairs-with-smallest-sums

思路

暴力枚举+优先队列。

暴力枚举所有可能的点对，放入最小堆并取出前k个——会超时超内存

优化 : 两个数组按下标枚举时均只最多枚举前k个，按topK的算法维护一个大顶堆。

LC215 数组中的第K个最大元素 [中等]

给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

思路一

直接调用O(n lg n)的排序算法排序

思路二

经典topK问题，维护一个大小为K的小顶堆。

如果堆大小不够K，则新来元素直接入堆。否则将堆顶元素和新元素比较，如果新元素更大，则把堆顶元素弹出，新元素入堆。最后留在堆里的就是前K大的所有元素，而堆顶元素就是第K大的元素。

LC692 前Ｋ个高频单词［中等］

给一非空的单词列表，返回前 k 个出现次数最多的单词。

返回的答案应该按单词出现频率由高到低排序。如果不同的单词有相同出现频率，按字母顺序排序。

思路

用HashMap记录单词的输出频率，维护大小为K的堆。注意比较的条件。

* LC295 数据流的中位数 [困难]

中位数是有序列表中间的数。如果列表长度是偶数，中位数则是中间两个数的平均值。

设计一个支持以下两种操作的数据结构：

  • void addNum(int num) - 从数据流中添加一个整数到数据结构中。
  • double findMedian() - 返回目前所有元素的中位数。

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/find-median-from-data-stream

思路

维护一个大顶堆maxPQ和小顶堆minPQ，分别维护前一半小的数据和后一半小的数据，并保持两堆元素数量相同或者minPQ比maxPQ至多多1个元素。每次新加入一个元素时考虑和两个堆堆顶元素的大小关系再决定加入到哪个堆中。中位数可以由两个堆的堆顶元素得到。

* LC264 丑数II [中等]

给你一个整数 n ，请你找出并返回第 n 个 丑数 。

丑数 就是只包含质因数 2、3 和/或 5 的正整数。

思路一

三指针，见 ”队列练习“一节

思路二

小顶堆+广搜

为避免扩展重复的结点，我们规定由当前结点x扩展后续结点的规则是：在2、3、5这三个素因子中，x只能乘以大于等于自己含有的这些因子中最大值的因数

LC1753 移除石子的最大得分 [中等]

你正在玩一个单人游戏，面前放置着大小分别为 a、b 和 c 的 三堆 石子。

每回合你都要从两个 不同的非空堆 中取出一颗石子，并在得分上加 1 分。当存在 两个或更多 的空堆时，游戏停止。

给你三个整数 a 、b 和 c ，返回可以得到的 最大分数 。

提示：

  • 1 <= a, b, c <= 105

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/maximum-score-from-removing-stones

思路

贪心+优先队列。尽可能避免空堆的出现，每次都从元素最多的堆里拿。

动态维护a,b,c三个数字构成的大顶堆。每次取出最大的两个数字，各自减一并让计数加一，减为0时不再入队。反复直到无法再找到两个数字拿来减。

总结

熟悉topk的经典解法

维护一个大小为k的优先队列。堆没有放满的情况下，直接往堆里面添加，直到添加到K的大小。堆的大小等于K之后，每次存放数的时候，和堆顶比较，如果堆顶的元素大于当前的数，那么出队

例题如LC215 373等

topK的一些变形

维护中位数、1/3位数等都是变相地求topK，思路可以往topK靠。

求只含某些因子的前n个数

LC264、LC313

可以是优先队列的解法，也可以是多路归并的做法

技巧

暴力枚举时可以粗略估计需要获取的数据量，这样减少数据规模可以避免超时超内存 如LC373暴力时的两层for循环的条件

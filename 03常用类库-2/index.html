03 常用类库 -2

类集

Collection

单值集合的顶级父接口

Collection接口的方法

在开发中一般不会直接使用Collection接口，而使用其子接口: List 、 Set

List 中存的元素允许重复，Set中的元素不允许重复

增 add / addAll

删 remove / removeAll / clear

查 contains / iterator / retainAll

类型转换 toArray

List

List 是 Collection的子接口，里面的元素允许重复

这里的List 的remove方法做了重载，根据下标来获取对象。

List 的实现类 : ArrayList (95%) 、Vector(4%) 、 LinkedList (1%)

ArrayList 是动态扩容 ， LinkedList是用链表实现

增 add / addAll (有重写，根据下标)

删 remove (有重写，根据下标)

改 set

查 get / indexOf / lastIndexOf

ArrayList

使用数组结构

Vector

使用数组结构，是同步的，若不考虑线程安全，则建议使用ArrayList代替Vector

大部分方法与ArrayList相同

新增的方法如下

LinkedList

使用双向链表结构，可以作为栈或队列

Iterator

对迭代器修改会修改对应的List对象

hasNext 、next 、 remove

创建迭代器后iterator并不是指向arr的首个元素而是指在首元素之前，需要调用iterator.next()才能使其指向arr的首个元素

ListIterator

Iterator的子接口

增加了 add 、 hasPrevious 、previous 、set 等方法

foreach

增强for循环，用于迭代数组或集合(Collection)

代码见上

Set

不包含重复元素的集合

继承自Collection接口，没有增加多少方法，方法与Collection基本相同

没有办法通过get来获取元素，需要用iterator访问或toArray转成数组再访问

HashSet

散列存放 ，无序存放， 基于HashMap实现

TreeSet

有序二叉树存储，基于TreeMap实现

此类的iterator方法返回的迭代器是快速失败的

如要自定义排序规则，则需要实现Comparable接口，重写compareTo方法(返回正/负/零)

Map

Map<K,V>

多值存储，存储一个个键值对数据，键不可重复

Set集合借助Map的键来存储数据从而保证数据不重复的性质

clear 清空

keySet 返回键的集合。

values 返回值的集合

遍历需先通过keySet获取键的集合，再通过键来获取值

get 获取键对应的值

put 存储一组键值对，若键已存在则会替换并返回旧值，否则返回null

remove 删除并取出。可以只根据键来删，也可以根据键值对来删除（匹配才做删除），都返回键对应的值或者不存在时返回null

containsKey 判断键是否存在

size 获取键值对数量

HashMap

哈希表

对象数组+链表/红黑树的方式存储。根据对象hashCode和散列函数（模16）来计算其在对象数组的下标，如果发生冲突，则采用链表的方式放在同一个哈希桶里

当哈希桶里的数据量大于8时链表会转成红黑树

当哈希桶里的数据量减少到6时，红黑树会转回链表

初始桶数量是16，散列因子是 0.75 （即若哈希表超过75%被使用了则会重新散列）

散列因子越大越节省空间，但查询效率更低

HashMap 、 Hashtable 、ConcurrentHashMap、 TreeMap、LinkedHashMap 的用法相同

TreeMap 对于自定义类型需要额外自定义排序方法

HashMap 、 Hashtable 、ConcurrentHashMap的区别

区别在于线程安全与否

HashMap 线程不安全，效率高。 同时取数据

Hashtable 线程安全，效率低。 排队取数据

ConcurrentHashMap 采用分段锁机制，保证线程安全且效率比较高。 访问哈希数组时仅在下标相同时排队，下标不相同则同时访问

存储自定义对象

当自定义对象作为HashMap的键时，修改键的原对象后再通过原对象在HashMap中查找可能找不到对应的值。

但是如果在后续哈希表重新散列了（比如插入较多的键值对），则可以根据新的对象的内容查找到。

不建议将需要修改的自定义对象作为HashMap的键

键比较的过程：先计算hashCode在HashMap中找到对象，在调用equal判断相等

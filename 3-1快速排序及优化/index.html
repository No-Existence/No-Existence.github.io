快速排序及优化

理论

快速排序

STL中的快速排序优化

单边递归

降低栈的空间

无监督 Partition

减少 l<=r 的判断

调整枢轴量的选取方法

三值取中 或者 随机选取。防止从 O(nlog n) 降低到 O(n^2)

结合堆排序和插入排序

堆排序在给定任意数据集的时间复杂度都是 O(nlog n) ，不会像快排一样在数据集有序时退化为O(n^2)。所以在快排的递归树深度较深时会即使转换为堆排序。

插入排序利用了数组的有序程度。在数组长度小于等于一定长度（16）时，STL会从快速排序改为插入排序。

LC912 数组排序 [中等]

给你一个整数数组 nums，请你将该数组升序排列。

代码

剑指offer 21 调整数组顺序使奇数位于偶数前面 [简单]

输入一个整数数组，实现一个函数来调整该数组中数字的顺序，使得所有奇数位于数组的前半部分，所有偶数位于数组的后半部分。

思路

借助Partition的实现思路。Partition将所有比pivot小的放在数组左边，比pivot大的放在数组右边。

类似的，该题要求的是将所有奇数放数组左边，所有偶数放在数组右边。

* LC148 排序链表 [中等]

给你链表的头结点 head ，请将其按 升序 排列并返回 排序后的链表 。

进阶：

你可以在 O(n log n) 时间复杂度和常数级空间复杂度下，对链表进行排序吗？

提示：

  • 链表中节点的数目在范围 [0, 5 * 104] 内
  • -105 <= Node.val <= 105

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/sort-list

思路

借助快排的思路，选取一个枢轴量pivot，将链表中所有结点值小于pivot的结点构成一个链表L1，剩余结点构成一个链表L2，分别对L1和L2排序，最后拼接起来。注意枢轴量的选取，要避免快排退化到O(n^2) 所以这里我们选取链表中最大值和最小值的中间值。

容易在快速排序优化方面被卡住

LC75 颜色分类 [中等]

给定一个包含红色、白色和蓝色，一共 n 个元素的数组，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

此题中，我们使用整数 0、 1 和 2 分别表示红色、白色和蓝色。

提示：

  • n == nums.length
  • 1 <= n <= 300
  • nums[i] 为 0、1 或 2

进阶：

  • 你可以不使用代码库中的排序函数来解决这道题吗？
  • 你能想出一个仅使用常数空间的一趟扫描算法吗？

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/sort-colors

思路

荷兰国旗问题。由于三类数值的值是唯一的，只用确定边界的下标，然后按顺序填充数值即可。而左边界下标可以通过统计0的个数，右边界的下标为数组长度减去2的个数

剑指 Offer 40. 最小的k个数 [简单]

输入整数数组 arr ，找出其中最小的 k 个数。例如，输入4、5、1、6、2、7、3、8这8个数字，则最小的4个数字是1、2、3、4。

思路

Partition函数能确定保证枢轴量放在按大小排序好的正确位置，且左边都比枢轴量小，右边都比枢轴量大或相等。

LC394 字符串解码

给定一个经过编码的字符串，返回它解码后的字符串。

编码规则为: k[encoded_string]，表示其中方括号内部的 encoded_string 正好重复 k 次。注意 k 保证为正整数。

你可以认为输入字符串总是有效的；输入字符串中没有额外的空格，且输入的方括号总是符合格式要求的。

此外，你可以认为原始数据不包含数字，所有的数字只表示重复的次数 k ，例如不会出现像 3a 或 2[4] 的输入。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/decode-string

思路

括号匹配借助map来映射一对括号的下标。在递归时可借助该映射快速找到左括号对应的右括号，从而确定子问题对应的字符串

LC470 用 Rand7() 实现 Rand10() [中等]

已有方法 rand7 可生成 1 到 7 范围内的均匀随机整数，试写一个方法 rand10 生成 1 到 10 范围内的均匀随机整数。

不要使用系统的 Math.random() 方法。

提示:

  1. rand7 已定义。
  2. 传入参数: n 表示 rand10 的调用次数。

进阶:

  1. rand7()调用次数的期望值是多少 ?
  2. 你能否尽量少调用 rand7() ?

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/implement-rand10-using-rand7

思路

由两个rand7随机生成一个两位数满足十位是12且个位是15的概率是10/49。假设我们已经找到这样的一个数，那从中抽取任意一个数的概率就是1/10

LC11 盛最多水的容器 [中等]

给你 n 个非负整数 a1，a2，…，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂直线 i 的两个端点分别为 (i, ai) 和 (i, 0) 。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

说明：你不能倾斜容器。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/container-with-most-water

思路

最终我们要求解的是 (i-j)*min{height[i],height[j]} (i>j) 的最大值。变化的量有两部分 i-j 和min{height[i],height[j]} 。考虑固定一方然后求解另一方的最大值后再求所有这样解的最大值。

我们将垂直线按高度从小到大排序，每次取出最矮的垂直线k作为容器的一边，这样能保证 min{height[i],height[j]} 的值一定是该垂直线的高度height[k]，然后找出距离该垂直线最远的线计算 (i-j)*height[k]，然后将直线k删除，维护垂直线集合的最左和最右位置的指针。
